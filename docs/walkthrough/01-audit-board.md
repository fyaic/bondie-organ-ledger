# 审计看板页用户指南

看板页是 OrganLedger 的主工作台。它把分散的文件改动整理成一屏概览，让用户先看“最近有没有异常变化”，再决定要不要去日志或文件树里继续看。

你打开这一页时，不需要先懂 git 或日志格式。页面会把关键问题摆在上面：有多少改动、严重度如何、涉及哪些系统。

![看板总览](assets/01-board-overview.png)

## 1. 先看整体风险

顶部概览区像一个仪表盘：

- **待确认**：被治理门扣住、等你批准或拒绝的改动数量。
- **改动数**：当前时间范围内记录到的变更数量。
- **涉及文件**：这批变更影响了多少个文件。
- **严重度**：帮助你优先关注 high 或 critical。
- **系统**：区分改动来自 OpenClaw、Hermes 或其他目标。

这个区域的目的不是展示更多数据，而是让你在几秒钟内判断：今天的器官变化是否平静，还是需要继续追踪。

概览下方的**器官来源 / Sources** 表，列出每个被治理仓库的 remote、分支、HEAD 和上游 ahead/behind——回答“这些器官从哪来、和上游差多少”。

## 2. 用筛选把注意力收窄

页面顶部的筛选栏是看板页的“聚焦器”。当记录很多时，不需要从头翻到尾，可以直接按你的问题筛：

- 想看最近状况：保留“近 7 天”。
- 想查历史：切到“全部”。
- 想只看高风险：把严重度切到 high 或 critical。
- 想区分来源：选择“上游更新”或“本地改动”。
- 想看是谁触发：按“修改者”筛 IM 用户请求、agent 自主、本机或未知。
- 已经知道文件名：直接搜索路径片段或 change id。

这里的“修改者”更准确地说是“主使线索”。如果显示本机，意思是 OrganLedger 知道改动来自这台机器，但不会武断区分是你本人、Coding Agent，还是其他本机进程。

## 3. 点开单条变更看详情

在状态列（待确认 / 已观测 / 已批准 / 已拒绝）里点任意一张变更卡片，右侧会滑出详情抽屉。

![单条变更详情抽屉](assets/04-activity-day-drawer-detail.png)

抽屉把这条改动的全部证据摆清楚：状态、操作类型、系统、原因、**before / after 哈希**、git commit、session、author、时间。哈希链就是账本可信的根基——同一改动在 Windows 和 macOS 上会产出相同的哈希，任何篡改都能被 `verify-ledger` 查出。

如果这条改动落在“待确认”列（例如高风险删除被治理门扣住），看板不会替你处置，而是给出可复制的命令，由你在终端里 `organledger approve` / `reject`。**看板只读**——它组织证据，处置权始终在你手里。
