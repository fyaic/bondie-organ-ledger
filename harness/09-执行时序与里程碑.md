## Phase -1 环境探测（硬前置，见 INIT 第五节）
```bash
node --version                 # ≥ v24 ？
git --version
cd /c/Hello-World/organledger
git checkout -b feat/cross-platform
node --test test/*.test.ts     # 绿色基线！记录 pass 数到 99-偏差日志
ls .gitattributes              # 应不存在
grep -rn 'split("\\\\n")' src --include=*.ts   # CRLF 风险点清单
```
任一探测失败 → 记 TODO「已知阻塞」→ 启用替代（如 node<24 则先 `nvm use 24`；无法则记录并继续做不依赖运行时的改动如 .gitattributes/CI）→ **不停止**。

## 后置三轮（核心做完还有时间就进）
- **轮 A 修复阻塞**：清 TODO「已知阻塞」，逐个再试。
- **轮 B 代码审查**：`git diff` 逐行核对——① 没动 02 矩阵 C 区；② 没改 canonicalJson/账本序列化；③ CRLF 改动只在「解析外部/文件文本」处；④ 全量测试仍绿。
- **轮 C 扩展**：给 `DEV-README.md` 补「跨平台兼容性」小节（列已修项 + .gitattributes 说明 + CI 矩阵）；评估 xdg-open 防御是否顺手补。

## 停止条件
只有 **120 分钟到** 或 **人类喊停**。核心早完 → 进后置，别停下来问「接下来做什么」。
