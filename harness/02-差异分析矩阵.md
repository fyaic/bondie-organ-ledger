# 02-差异分析矩阵 —— Win vs Mac 逐项对比

> 来源：参谋长对 `src/**` + `test/**` 的逐文件审计（非 grep，读过代码）。
> 「支持度」列：❌=有真实 bug，⚠️=可加固，✅=已正确。

## A. 会破坏哈希链的差异（BLOCKER 区）

| 维度 | Windows 行为 | macOS 行为 | 支持度 | 根因 file:line |
|------|-------------|-----------|--------|---------------|
| 换行入库 | 默认 `core.autocrlf=true`，checkout 时 LF→CRLF | LF 原样 | ❌ | 缺 `.gitattributes`（仓库根） |
| 文件哈希 | `fileSha()` 读到 CRLF 版本 → hash≠HEAD(LF) | 一致 | ❌ | 级联自上（`util.ts:206`、`ledger.ts:70`、`normalizer.ts:126`） |
| git raw 解析 | `git show --raw` 可能带 CRLF，`split("\n")` 残 `\r` → 字段错位 | 一致 | ❌ | `backfill.ts:190` |

## B. 可加固的差异（MAJOR / MINOR 区）

| 维度 | Windows | macOS | 支持度 | file:line |
|------|---------|-------|--------|-----------|
| 进程信号 | **无 SIGTERM**；有 SIGINT/SIGBREAK | 有 SIGTERM/SIGINT | ⚠️ | `cli/index.ts:348-349` |
| python 探测 | `python`（可能不在 PATH） | `python3` | ⚠️ | `test/hermes.test.ts:23` |
| CI 覆盖 | 未测 | 未测 | ⚠️ | `.github/workflows/ci.yml:10`（仅 ubuntu） |
| UNC 路径 | `\\server\share` | 无 | ⚠️(次) | `reveal.ts:55`（isAbsolute 已兜底，仅补注释） |
| xdg-open | N/A | N/A（走 open） | ⚠️(次) | `cli/index.ts:65`（仅 Linux 最小系统，加一行防御） |
| 锁文件删除 | 文件占用时 `rmSync` 可能 EACCES | 正常 | ✅(已缓解) | `daemon.ts:58` —— acquireLock 有夺锁逻辑，**勿动** |
